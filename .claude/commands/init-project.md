Inicializa un CLAUDE.md en un proyecto combinando el Common Layer (universal) con el Project Layer (específico).

## Argumentos

$ARGUMENTS — Ruta al directorio del proyecto. Si no se proporciona, usa el directorio actual.

## Instrucciones

### 1. Verificar estado actual

Comprueba si ya existe un CLAUDE.md en el directorio target:
- Si existe: pregunta al usuario si quiere sobrescribir, hacer merge, o cancelar.
- Si no existe: continúa.

### 2. Leer los templates

Lee los siguientes archivos del framework:
- `templates/common-layer.md` — la capa universal
- `templates/project-layer.md` — la capa específica del proyecto

### 3. Detectar el proyecto (si es existente)

Si el directorio target ya tiene código:

1. Lee la estructura del proyecto (`ls -la`, `find` para archivos clave)
2. Detecta el stack: busca `package.json`, `pyproject.toml`, `Cargo.toml`, `Gemfile`, `go.mod`, `Dockerfile`, `docker-compose.yml`
3. Detecta el framework: busca imports/configs de frameworks conocidos
4. Detecta comandos de test/lint/build existentes en scripts, Makefile, CI configs
5. Pre-rellena lo que puedas en el Project Layer:
   - Stack table con las tecnologías detectadas
   - Commands section con los comandos encontrados
   - Quality Gates con los test/lint/type commands reales

### 4. Componer el CLAUDE.md

Combina ambas capas en un solo archivo:

```markdown
# [Project Name]

[Auto-detected or user-provided description]

---

[Common Layer content — completo, sin modificaciones]

---

[Project Layer content — pre-rellenado donde sea posible, templates vacíos para el resto]

---

<!-- Generated by Operational AI Workflows framework on [date].
     Common Layer: auto-updated when framework evolves.
     Project Layer: maintained via /session-review and manual updates.
     See: protocols/internal-maintenance.md -->
```

### 5. Instalar hooks de automatización

Copia los hooks del framework al proyecto target:

1. Crea `.claude/hooks/` en el directorio target si no existe
2. Copia `templates/hooks/session-review-hook.sh` a `.claude/hooks/session-review-hook.sh`
3. Copia `templates/hooks/context-watchdog.sh` a `.claude/hooks/context-watchdog.sh`
4. Hazlos ejecutables (`chmod +x`)
5. Si existe `.claude/settings.json`: haz merge de la configuración de hooks (no sobrescribir)
6. Si no existe: copia `templates/hooks/settings.json.template` como `.claude/settings.json`

También copia skills y commands del framework:
1. Crea `.claude/skills/session-review/` en el directorio target
2. Copia `.claude/skills/session-review/SKILL.md`
3. Copia `.claude/commands/context-check.md` a `.claude/commands/context-check.md`

### 6. Escribir el archivo

Escribe el CLAUDE.md compuesto en la raíz del directorio target.

### 7. Informar al usuario

Muestra:
- Qué secciones se pre-rellenaron automáticamente
- Qué secciones necesitan ser completadas manualmente
- Sugiere ejecutar `/bootstrap-intelligence` si el proyecto tiene historial de sesiones en `.claude/`

### 8. Si el proyecto tiene sesiones previas

Si detectas un directorio `.claude/` con sesiones o historial:
```
Sugiere: "Este proyecto tiene historial de sesiones. Ejecuta /bootstrap-intelligence para extraer
patrones y gotchas automáticamente."
```
